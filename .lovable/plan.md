

# Seed Blood Bank Data & Replace Hardcoded Analytics

## Overview

Two parts:
1. **Insert realistic test data** into `blood_donors`, `blood_donations`, `blood_inventory`, and `blood_requests` via an edge function (since we need to bypass RLS and use the org/branch IDs)
2. **Rewrite the Analytics page** to query real data from Supabase instead of using hardcoded `SEED_DATA`

## Known Constants

- Organization ID: `a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11`
- Branch ID: `b1eebc99-9c0b-4ef8-bb6d-6bb9bd380a11`
- Patient IDs available: `21eebc99-...` through `25eebc99-...` (5 patients)
- Staff user for `requested_by`: `00000000-0000-0000-0000-000000000030`
- Tables currently have 0 rows
- `donor_number`, `donation_number`, `unit_number`, `request_number` are auto-generated by triggers

## Part 1: Edge Function `seed-blood-bank`

### New File: `supabase/functions/seed-blood-bank/index.ts`

Creates realistic demo data using the service role key (bypasses RLS):

**20 Blood Donors** across all 8 blood groups with realistic demographics:
- Mix of genders, ages 20-55, weights 55-95kg
- 18 active, 1 deferred, 1 inactive
- Varying `last_donation_date` and `total_donations` counts
- Realistic hemoglobin levels (12-16 g/dL)

**60 Blood Donations** spread across the last 12 months:
- Linked to the 20 donors (2-4 each)
- Mix of statuses: ~50 completed, 5 rejected, 5 processing
- Realistic vitals (BP, pulse, temperature, hemoglobin)
- Various donation types (mostly whole_blood, some platelet, plasma)

**80 Blood Inventory Units** with varied statuses and expiry dates:
- Linked to completed donations where possible
- Component types: whole_blood, packed_rbc, fresh_frozen_plasma, platelet_concentrate, cryoprecipitate
- Statuses: ~50 available, 10 issued, 5 transfused, 5 expired, 5 quarantine, 5 reserved
- **3 units expiring within 3 days** (to trigger the expiry alert banner)
- **5 units expiring within 7 days** (for the inventory filter)
- Collection dates spanning 12 months
- Storage locations: Fridge A, Fridge B, Freezer 1, Platelet Agitator
- All tests marked as passed for available units

**15 Blood Requests** linked to existing patients:
- Mix of priorities: 8 routine, 4 urgent, 3 emergency
- Statuses: 5 pending, 3 processing, 2 ready, 3 completed, 2 cancelled
- Various blood groups and component types
- Clinical indications: surgery, anemia, trauma, etc.

### Registration in `supabase/config.toml`

Add `[functions.seed-blood-bank]` with `verify_jwt = false`.

## Part 2: Analytics Page -- Real Queries

### New Hook: `useBloodBankAnalytics` in `src/hooks/useBloodBank.ts`

Fetches aggregated data from the 4 tables:

1. **Monthly collection vs consumption**: Query `blood_donations` (status=completed, grouped by month) and `blood_inventory` (status in issued/transfused, grouped by month)
2. **Blood group distribution**: Query `blood_inventory` grouped by `blood_group`, count per group
3. **Donation trends**: Query `blood_donations` grouped by month, count completed vs rejected
4. **Component breakdown**: Query `blood_inventory` grouped by `component_type`, count units
5. **Summary stats**: Total collections this month, total consumed this month, calculate rates

### Modified File: `src/pages/app/blood-bank/BloodBankAnalyticsPage.tsx`

- Remove the `SEED_DATA` constant entirely
- Import and use `useBloodBankAnalytics` hook
- Add loading skeletons while data loads
- Transform query results into chart-compatible format
- Keep the same chart configs and layout
- Show "No data" state if tables are empty
- Summary stats cards use real computed values

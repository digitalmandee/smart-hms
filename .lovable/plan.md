

# Implementation Plan: Journal Entry Form, Bank Account Form, General Ledger Fix

## Overview

Three tasks:
1. Create a Journal Entry form page at `/app/accounts/journal-entries/new`
2. Create a Bank Account form page at `/app/accounts/bank-accounts/new`
3. Fix General Ledger dropdown to only show Level 4 posting accounts

No database changes needed -- all tables already exist with the correct schema.

---

## Task 1: Journal Entry Form Page

### New File: `src/pages/app/accounts/JournalEntryFormPage.tsx`

A full-page form with:

**Header section:**
- Entry Date (date picker, defaults to today)
- Description (text input)
- Notes (optional textarea)
- Reference Type dropdown (manual, invoice, shipment, stock_adjustment)

**Line items section (dynamic multi-line):**
- Each line: Account picker (L4 only via `AccountPicker` with filtering), Description, Debit amount, Credit amount
- "Add Line" button to add rows
- "Remove" button per row (trash icon)
- Only one of debit/credit can be non-zero per line (auto-clear the other)

**Footer/validation:**
- Running totals for Debit and Credit columns
- Balance indicator: green checkmark when totals match, red warning when unbalanced
- Buttons: "Save as Draft" (inserts with `is_posted = false`) and "Post Entry" (inserts with `is_posted = true`, sets `posted_at` and `posted_by`)
- "Post Entry" disabled when totals don't match or no lines exist
- Minimum 2 lines required

**Data flow:**
1. Insert into `journal_entries` (entry_number auto-generated by DB trigger `generate_journal_entry_number`)
2. Insert all lines into `journal_entry_lines`
3. DB trigger `update_account_balance` auto-updates account balances
4. DB trigger `check_posting_account` prevents posting to header accounts
5. Navigate to the new entry's detail page on success

**Account selection:** Reuse the existing `AccountPicker` component but pass a filter to only show L4 accounts. The `AccountPicker` already uses `useAccounts({ isActive: true, category })` -- we need to filter `is_header = false` in the returned data. The `useAccounts` hook doesn't have an `isHeader` filter, so we'll filter client-side in the form (accounts already have `is_header` in the data).

### Modified File: `src/App.tsx`
- Add import for `JournalEntryFormPage`
- Add route: `accounts/journal-entries/new` (before the `:id` route on line 991)

## Task 2: Bank Account Form Page

### New File: `src/pages/app/accounts/BankAccountFormPage.tsx`

A form with fields matching the `bank_accounts` table:
- Bank Name (required text)
- Account Number (required text)
- Account Holder Name (optional text)
- Account Type (select: savings, current, fixed_deposit)
- Branch (select from `branches` table)
- IFSC Code (optional)
- SWIFT Code (optional)
- Opening Balance (number, default 0)
- Is Default (checkbox)
- Link to Chart of Accounts account (optional `AccountPicker` for the `account_id` FK)

On submit: insert into `bank_accounts` with `organization_id` and `created_by` from auth context. Navigate back to bank accounts list.

### Modified File: `src/App.tsx`
- Add import for `BankAccountFormPage`
- Add route: `accounts/bank-accounts/new` (after line 998)

## Task 3: Fix General Ledger Dropdown

### Modified File: `src/pages/app/accounts/GeneralLedgerPage.tsx`

Line 30: Change `useAccounts()` to `useAccounts({ isActive: true })` and filter the dropdown to exclude header accounts:

```typescript
const { data: allAccounts = [] } = useAccounts({ isActive: true });
const accounts = allAccounts.filter(a => !a.is_header);
```

This ensures only Level 4 posting accounts appear in the account selection dropdown.

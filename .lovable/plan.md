

# Pharmacy POS Session Closing → Journal Entry Integration + Closings Report Screen

## Current Gap
POS session open/close works, but **does not post to the accounting general ledger**. When a pharmacy session closes, the system should create a journal entry (DR Cash, CR Pharmacy Revenue) for the total sales, matching how vendor payments and other financial events auto-post. There is also no dedicated "Pharmacy Closings" screen showing a history of all closings with downloadable reports.

## Implementation

### 1. Post Journal Entry on Session Close
**File: `src/hooks/usePOSSessions.ts`** — in `useCloseSession` mutation

After updating the session status to "closed", create a journal entry:
- Look up or create accounts using Supabase RPC `get_or_create_default_account`:
  - `CASH-001` (Cash in Hand) — asset
  - `REV-PHARM-001` (Pharmacy Revenue) — revenue
- Insert `journal_entries` record with `reference_type: 'pos_session'`, `reference_id: sessionId`, `is_posted: true`
- Insert two `journal_entry_lines`:
  - **Debit**: Cash account → `totalSales` amount
  - **Credit**: Pharmacy Revenue account → `totalSales` amount
- If cash difference exists, create an additional over/short line using account `EXP-CASH-DIFF-001` (Cash Over/Short — expense)
- Store `journal_entry_id` reference on the session (or just link via reference_id)

Pattern follows existing `useVendorPayments.ts` journal posting logic exactly.

### 2. New Pharmacy Closings Page
**New file: `src/pages/app/pharmacy/PharmacyClosingsPage.tsx`**

A dedicated closings report screen at `/app/pharmacy/closings`:
- Date range filter (today, this week, this month, custom)
- Table: Date, Session #, Cashier, Opening, Total Sales, Cash/Card/Mobile breakdown, Expected Cash, Closing Balance, Difference, Status
- Summary cards at top: Total Sessions, Total Revenue, Total Cash Difference, Avg Session Duration
- Each row clickable → navigates to existing session detail page
- **PDF Download button** using `ReportExportButton` for the filtered data
- Color-coded difference column (green = balanced, red = short, blue = over)

### 3. Add Route & Navigation
**File: `src/App.tsx`** — add route `/app/pharmacy/closings`
**Sidebar** — add "Closings" link under Pharmacy section

### 4. Translations

## Technical Details
- Journal entry number auto-generated by existing `generate_journal_entry_number` trigger
- Uses `get_or_create_default_account` RPC to ensure accounts exist without manual setup
- Follows the exact same pattern as vendor payment journal posting in `useVendorPayments.ts`
- Only posts journal when `totalSales > 0` to avoid empty entries

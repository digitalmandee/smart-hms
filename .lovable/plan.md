

# PO → GRN → GL Journal Posting Fix

## Problem
When a GRN is "Posted", it only changes the status to `posted` — it does **not** create any journal entries in the General Ledger. The documented flow (DR Inventory Asset / CR Accounts Payable) is not implemented. This means:
- No GL impact from goods receipt
- Accounts Payable doesn't reflect vendor obligations
- Inventory asset value isn't tracked in the ledger

## Current Flow
```text
PO (approved → ordered) → GRN (draft → verified → posted)
                                    ↑ stock updated    ↑ status only, NO GL
```

## Solution
Update `usePostGRN` in `src/hooks/useGRN.ts` to create a journal entry when posting:

1. **Fetch GRN with items** to calculate total accepted value
2. **Look up GL accounts** — find Inventory Asset and Accounts Payable accounts by account code (e.g., `INV-001` / `AP-001` or by account type category)
3. **Create journal entry** with `reference_type: 'grn'`, `reference_id: grn.id`
4. **Create journal entry lines**:
   - DR: Inventory Asset account → total GRN value
   - CR: Accounts Payable account → total GRN value
5. **Update account balances** on both accounts
6. Mark GRN status as `posted`

### Account Lookup Strategy
Query `accounts` table for accounts with category/type matching inventory asset and accounts payable. If not found, show error asking user to set up Chart of Accounts first.

### Journal Entry Details
- `entry_number`: auto-generated by DB trigger
- `entry_date`: GRN received_date
- `reference_type`: `'grn'`
- `reference_id`: GRN id
- `description`: `"GRN {grn_number} — Goods received from {vendor_name}"`
- `is_posted`: true
